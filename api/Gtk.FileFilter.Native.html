add_custom(IntPtr, FileFilterFlags, FileFilterFunc, IntPtr, DestroyNotify)

Adds rule to a filter that allows files based on a custom callback function. The bitfield @needed which is passed in provides information about what sorts of information that the filter function needs; this allows GTK+ to avoid retrieving expensive information when it isn’t needed by the filter.

new()

Creates a new #GtkFileFilter with no rules added to it. Such a filter doesn’t accept any files, so is not particularly useful until you add rules with gtk_file_filter_add_mime_type(), gtk_file_filter_add_pattern(), or gtk_file_filter_add_custom(). To create a filter that accepts any file, use: |[ GtkFileFilter filter = gtk_file_filter_new (); gtk_file_filter_add_pattern (filter, ""); ]|
